# BI Platform — Презентация для заказчика

## 1. Общее описание системы

Корпоративная система бизнес-аналитики (BI), которая объединяет данные из amoCRM, 1С и Google Sheets в единое хранилище с интерактивными дашбордами и SQL-аналитикой.

**Стек технологий:**
- PostgreSQL 16 — хранилище данных (DWH)
- Metabase — визуализация и дашборды
- FastAPI — backend API и авторизация
- Next.js + Vercel — публичный веб-интерфейс
- Docker — контейнеризация всего стека

---

## 2. Архитектура данных

```
amoCRM ──┐
1С      ──┼── ETL (Python) ── PostgreSQL DWH ── Metabase / Next.js
Google   ──┘                    (Star Schema)
```

### Структура DWH (Star Schema)

**Измерения (Dimensions):**
- `dim_date` — календарная шкала (5 лет)
- `dim_counterparty` — контрагенты (клиенты, поставщики, партнеры)
- `dim_product` — продукты и услуги
- `dim_department` — отделы (Продажи, Маркетинг, Закупки, Бухгалтерия, IT)
- `dim_employee` — сотрудники
- `dim_account` — план счетов для P&L (Revenue, COGS, OPEX, Tax)
- `dim_currency` — валюты (RUB, USD, EUR, CNY)

**Факты (Facts):**
- `fact_transactions` — финансовые транзакции из 1С + ручной ввод
- `fact_sales` — сделки из amoCRM
- `fact_expenses` — расходы из Google Sheets + ручной ввод
- `fact_crm_leads` — лиды из amoCRM
- `fact_exchange_rates` — курсы валют ЦБ РФ (автозагрузка)

## 4. Функциональные модули

### A. Дашборд P&L, Cashflow, KPI

Интерактивные графики Metabase с фильтрами по дате, отделу, менеджеру.

### B. Обзор базы данных

Все таблицы DWH доступны для просмотра в браузере:
- Просмотр любой таблицы с пагинацией
- Фильтрация по схемам (public, staging, analytics)
- Прямой доступ к фактам и измерениям

### C. SQL Query Editor

Полноценный SQL-редактор для аналитиков:
- Выполнение произвольных SELECT-запросов
- Просмотр результатов в табличном виде
- Доступ к аналитическим вьюшкам:
  - `vw_pnl_waterfall` — P&L с маржинальностью
  - `vw_cashflow_monthly` — ежемесячный cashflow
  - `vw_sales_funnel` — воронка продаж
  - `vw_kpi_sales_managers` — KPI менеджеров
  - `vw_expenses_breakdown` — детализация расходов

### D. Ручной ввод данных

Формы для оперативного ввода без доступа к БД:
- **Новый расход** — категория, статья, сумма, валюта, дата
- **Новая транзакция** — дата, сумма, счет, описание
- Данные сразу попадают в `fact_expenses` / `fact_transactions`
- Помечаются как `is_manual_entry = true`

## 9. Рекомендации по развитию

### Для production-развертывания:
1. **Облачный VPS** (Yandex Cloud / VK Cloud / Selectel) — разместить Metabase и PostgreSQL на сервере в РФ
2. **Заменить localtunnel** на постоянный домен с SSL
3. **Настроить ежедневный ETL** по расписанию (cron + Docker)
4. **Row-Level Security** в Metabase для ограничения данных по ролям
5. **Алерты** — уведомления в Telegram при отклонении KPI от плана

### Оценка стоимости production:
- VPS 4 CPU / 8 RAM / 100 GB SSD: ~3,000–5,000 ₽/мес
- Домен + SSL: ~1,000 ₽/год
- Metabase + PostgreSQL: open-source, бесплатно
